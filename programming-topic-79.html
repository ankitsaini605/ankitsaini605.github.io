📌 Greedy Algorithms

Greedy Algorithms build up a solution piece by piece, always choosing the next piece that offers the most immediate benefit. They are efficient and easy to implement but don't always yield the optimal solution.

⚙️ Greedy Strategy

  • Make a greedy choice
  • Solve the subproblem
  • Combine solutions
  • Check if the local optimum leads to a global optimum

📊 Popular Examples

  • Activity Selection Problem
  • Fractional Knapsack
  • Job Scheduling Problem
  • Huffman Coding
  • Minimum Number of Coins

💻 Code Example: Activity Selection


bool activitySelection(vector<pair<int,int>> activities) {
  sort(activities.begin(), activities.end(), 
       [](auto &a, auto &b){ return a.second < b.second; });

  int count = 1;
  int end = activities[0].second;

  for (int i = 1; i < activities.size(); ++i) {
    if (activities[i].first >= end) {
      ++count;
      end = activities[i].second;
    }
  }

  return count;
}

📈 Complexity

  • Time: O(n log n) for sorting
  • Space: O(1) (greedy is generally space efficient)

✅ Real-Life Applications

  • Data compression using Huffman encoding
  • Task scheduling in operating systems
  • Resource allocation problems
Tip: Greedy algorithms work best when the problem exhibits the Greedy Choice Property and Optimal Substructure.
