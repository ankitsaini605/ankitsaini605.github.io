🔢 Radix Sort Algorithm

Digit-wise sorting approach for non-comparison-based sorting.

📘 What is Radix Sort?

Radix Sort is a non-comparative sorting algorithm that processes integers digit by digit. It uses a stable intermediate sort (like Counting Sort) to sort numbers based on each digit, starting from the least significant digit (LSD).

✨ Features:

  • Stable sorting algorithm
  • Efficient for integers with small digit counts
  • Best suited for fixed-length numeric data

🐍 Python Example:


def counting_sort_for_radix(arr, exp):
    n = len(arr)
    output = [0] * n
    count = [0] * 10

    for i in arr:
        index = (i // exp) % 10
        count[index] += 1

    for i in range(1, 10):
        count[i] += count[i - 1]

    i = n - 1
    while i >= 0:
        index = (arr[i] // exp) % 10
        output[count[index] - 1] = arr[i]
        count[index] -= 1
        i -= 1

    for i in range(n):
        arr[i] = output[i]

def radix_sort(arr):
    max_num = max(arr)
    exp = 1
    while max_num // exp > 0:
        counting_sort_for_radix(arr, exp)
        exp *= 10

arr = [170, 45, 75, 90, 802, 24, 2, 66]
radix_sort(arr)
print("Sorted:", arr)

💻 C++ Example:


#include <iostream>
#include <vector>
using namespace std;

void countingSort(vector<int>& arr, int exp) {
    vector<int> output(arr.size());
    int count[10] = {0};

    for (int i = 0; i < arr.size(); i++)
        count[(arr[i] / exp) % 10]++;

    for (int i = 1; i < 10; i++)
        count[i] += count[i - 1];

    for (int i = arr.size() - 1; i >= 0; i--) {
        int idx = (arr[i] / exp) % 10;
        output[count[idx] - 1] = arr[i];
        count[idx]--;
    }

    for (int i = 0; i < arr.size(); i++)
        arr[i] = output[i];
}

void radixSort(vector<int>& arr) {
    int maxVal = *max_element(arr.begin(), arr.end());
    for (int exp = 1; maxVal / exp > 0; exp *= 10)
        countingSort(arr, exp);
}

int main() {
    vector<int> arr = {170, 45, 75, 90, 802, 24, 2, 66};
    radixSort(arr);
    cout << "Sorted array: ";
    for (int num : arr) cout << num << " ";
    return 0;
}

🧠 Time & Space Complexity

  • Time: O(nk) where n is the number of elements and k is the number of digits
  • Space: O(n + k)

✅ Pros:

  • Efficient for sorting large number of small-digit integers
  • Faster than comparison-based algorithms in certain scenarios

⚠️ Cons:

  • Not suitable for floating-point numbers or strings without preprocessing
  • Uses additional memory for intermediate sorting
