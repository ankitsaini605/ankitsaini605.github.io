📈 Dynamic Programming (DP)

Dynamic Programming is a powerful technique used to solve complex problems by breaking them down into simpler subproblems. It stores the results of subproblems to avoid redundant computations, making algorithms significantly faster.

📚 Types of Dynamic Programming

  • Memoization (Top-Down): Recursively solve subproblems and store results.
  • Tabulation (Bottom-Up): Iteratively fill a table based on previous results.

🧠 Key Properties

  • Optimal Substructure: Problem can be broken into optimal subproblems.
  • Overlapping Subproblems: Same subproblems are solved multiple times.

💡 Common Dynamic Programming Problems

  1. Fibonacci Numbers
  2. Longest Common Subsequence (LCS)
  3. 0/1 Knapsack Problem
  4. Coin Change Problem
  5. Matrix Chain Multiplication
  6. Longest Increasing Subsequence

🧪 Example: 0/1 Knapsack (C++)


int knapsack(int W, int wt[], int val[], int n) {
    int dp[n+1][W+1];
    for (int i = 0; i <= n; i++) {
        for (int w = 0; w <= W; w++) {
            if (i == 0 || w == 0)
                dp[i][w] = 0;
            else if (wt[i-1] <= w)
                dp[i][w] = max(val[i-1] + dp[i-1][w - wt[i-1]], dp[i-1][w]);
            else
                dp[i][w] = dp[i-1][w];
        }
    }
    return dp[n][W];
}

🌐 Real-World Applications

  • Resource Allocation
  • Data Compression Algorithms (like LZW)
  • Spell Checking
  • AI & Game Strategy Calculations

🚀 Interview Tips

  • Identify overlapping subproblems and optimal substructure
  • Start with recursive → then memoize → then tabulate
  • Practice pattern-based problems (like DP on subsequence, grid, etc.)
Pro Tip: Always visualize the DP table/grid when stuck. It reveals patterns and transitions clearly!
