Introduction to Java Programming

Master the fundamentals of Java, one of the world's most popular and versatile programming languages.

Learn Basics See Examples

What is Java?

Java is a high-level, class-based, object-oriented programming language that was developed by James Gosling at Sun Microsystems (which has since been acquired by Oracle Corporation) and released in 1995. Java is designed to be platform-independent, following the principle of "Write Once, Run Anywhere" (WORA), which means that compiled Java code can run on all platforms that support Java without the need for recompilation.

Java applications are typically compiled to bytecode that can run on any Java Virtual Machine (JVM) regardless of the underlying computer architecture. This makes Java one of the most versatile and widely used programming languages in the world, powering everything from web applications and mobile apps to enterprise software and embedded systems.

Key Features of Java

Platform Independence

Java's "Write Once, Run Anywhere" capability allows code to run on any device with a Java Virtual Machine, regardless of the underlying hardware or operating system.

Security

Java provides multiple layers of security, including its bytecode verification process, strong memory management, and built-in exception handling to prevent system crashes.

Object-Oriented

Java follows object-oriented programming principles, organizing code into classes and objects that model real-world entities, promoting code reusability and maintainability.

Robust Networking

Java includes extensive libraries for networking, making it easy to develop distributed applications that communicate across networks.

Automatic Memory Management

Java's garbage collection automatically reclaims memory occupied by unused objects, reducing memory leaks and improving reliability.

Multithreading

Java provides built-in support for multithreaded programming, allowing developers to create applications that can perform multiple tasks simultaneously.

Java Architecture

Understanding Java's architecture is essential for grasping how Java programs work. The Java architecture consists of several key components that work together to enable the "Write Once, Run Anywhere" capability.

The Java Ecosystem

Java Development Kit (JDK)

The JDK is a software development environment used for developing Java applications. It includes the Java Runtime Environment (JRE), an interpreter/loader (Java), a compiler (javac), an archiver (jar), a documentation generator (Javadoc), and other tools needed for Java development.

Java Runtime Environment (JRE)

The JRE provides the libraries, Java Virtual Machine, and other components necessary to run Java applications. It's included in the JDK, but can also be installed separately if you only need to run Java programs, not develop them.

Java Virtual Machine (JVM)

The JVM is an abstract computing machine that enables a computer to run Java programs. It provides a runtime environment in which Java bytecode can be executed. The JVM is the cornerstone of the Java platform's "Write Once, Run Anywhere" principle.

Bytecode

Java source code is compiled into bytecode, which is a highly optimized set of instructions designed to be executed by the JVM. This intermediate representation is what makes Java platform-independent.

Java API (Application Programming Interface)

The Java API is a large collection of ready-made software components that provide many useful capabilities, such as graphical user interface (GUI) widgets, networking, and input/output functionality.

Your First Java Program

Hello World Example

Let's start with the traditional "Hello World" program to understand the basic structure of a Java application:

HelloWorld.java 
public class HelloWorld {
    public static void main(String[] args) {
        // This is a comment
        System.out.println("Hello, World!");
    }
}
Output:
Hello, World!

Code Explanation:

  • public class HelloWorld - Defines a class named HelloWorld. In Java, every application must have at least one class, and the class name must match the filename (HelloWorld.java).
  • public static void main(String[] args) - This is the main method, which is the entry point for any Java application. The Java runtime starts execution from this method.
  • // This is a comment - Comments in Java start with // for single-line comments or /* */ for multi-line comments.
  • System.out.println("Hello, World!"); - This statement prints the text "Hello, World!" to the console. The System.out is an output stream connected to the console, and println is a method that prints a line of text.

Pro Tip

In Java, every statement must end with a semicolon (;). Forgetting this is a common mistake for beginners. Also, Java is case-sensitive, so System is different from system.

Variables and Data Types

Java is a strongly-typed language, which means you must declare the type of a variable before using it. Here's an example that demonstrates various data types in Java:

DataTypes.java 
public class DataTypes {
    public static void main(String[] args) {
        // Primitive data types
        int age = 25;                  // Integer (whole number)
        double salary = 50000.50;      // Floating point number
        char grade = 'A';              // Single character
        boolean isEmployed = true;     // Boolean value
        
        // String is not a primitive type, it's a class
        String name = "John Doe";
        
        // Printing the values
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        System.out.println("Salary: $" + salary);
        System.out.println("Grade: " + grade);
        System.out.println("Employed: " + isEmployed);
        
        // Arithmetic operations
        int a = 10;
        int b = 5;
        System.out.println("a + b = " + (a + b));
        System.out.println("a - b = " + (a - b));
        System.out.println("a * b = " + (a * b));
        System.out.println("a / b = " + (a / b));
        System.out.println("a % b = " + (a % b));  // Modulus (remainder)
    }
}
Output:
Name: John Doe
Age: 25
Salary: $50000.5
Grade: A
Employed: true
a + b = 15
a - b = 5
a * b = 50
a / b = 2
a % b = 0

Java Data Types:

  • Primitive Types: These are the most basic data types available in Java.
    • byte: 8-bit integer (-128 to 127)
    • short: 16-bit integer (-32,768 to 32,767)
    • int: 32-bit integer (-2^31 to 2^31-1)
    • long: 64-bit integer (-2^63 to 2^63-1)
    • float: 32-bit floating-point
    • double: 64-bit floating-point
    • char: 16-bit Unicode character
    • boolean: true or false
  • Reference Types: These are more complex types like classes, interfaces, and arrays.
    • String: A sequence of characters
    • Array: A collection of similar type variables
    • Class: User-defined blueprint from which objects are created
    • Interface: A reference type that can contain only constants, method signatures, default methods, static methods, and nested types

Control Flow Statements

Java provides several control flow statements to control the flow of execution in your programs:

ControlFlow.java 
public class ControlFlow {
    public static void main(String[] args) {
        // If-else statement
        int score = 85;
        
        if (score >= 90) {
            System.out.println("Grade: A");
        } else if (score >= 80) {
            System.out.println("Grade: B");
        } else if (score >= 70) {
            System.out.println("Grade: C");
        } else if (score >= 60) {
            System.out.println("Grade: D");
        } else {
            System.out.println("Grade: F");
        }
        
        // Switch statement
        char grade = 'B';
        
        switch (grade) {
            case 'A':
                System.out.println("Excellent!");
                break;
            case 'B':
                System.out.println("Good job!");
                break;
            case 'C':
                System.out.println("Satisfactory");
                break;
            case 'D':
                System.out.println("Needs improvement");
                break;
            case 'F':
                System.out.println("Failed");
                break;
            default:
                System.out.println("Invalid grade");
                break;
        }
        
        // For loop
        System.out.println("Counting from 1 to 5:");
        for (int i = 1; i <= 5; i++) {
            System.out.print(i + " ");
        }
        System.out.println();
        
        // While loop
        System.out.println("Counting down from 5 to 1:");
        int count = 5;
        while (count > 0) {
            System.out.print(count + " ");
            count--;
        }
        System.out.println();
        
        // Do-while loop
        System.out.println("Do-while example:");
        int num = 1;
        do {
            System.out.print(num + " ");
            num++;
        } while (num <= 5);
        System.out.println();
    }
}
Output:
Grade: B
Good job!
Counting from 1 to 5:
1 2 3 4 5
Counting down from 5 to 1:
5 4 3 2 1
Do-while example:
1 2 3 4 5

Control Flow Statements:

  • If-else: Used for conditional execution based on a boolean expression.
  • Switch: Used for selecting one of many code blocks to be executed.
  • For loop: Used when you know how many times you want to execute a statement or a block of statements.
  • While loop: Repeats a statement or block of statements while a given condition is true. It tests the condition before executing the loop body.
  • Do-while loop: Similar to a while loop, but it tests the condition at the end of the loop body, which means the loop body will execute at least once.

Common Pitfall

Be careful with infinite loops! Always ensure that your loop condition will eventually become false, or use a break statement to exit the loop when needed.

Object-Oriented Programming in Java

Java is fundamentally an object-oriented programming (OOP) language. OOP is a programming paradigm based on the concept of "objects," which can contain data and code: data in the form of fields (often known as attributes or properties), and code in the form of procedures (often known as methods).

Classes and Objects

Here's an example of a simple class and how to create objects from it:

Person.java 
public class Person {
    // Fields (attributes)
    private String name;
    private int age;
    
    // Constructor
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    // Methods
    public void introduce() {
        System.out.println("Hello, my name is " + name + " and I am " + age + " years old.");
    }
    
    public void celebrateBirthday() {
        age++;
        System.out.println("Happy Birthday! Now I am " + age + " years old.");
    }
    
    // Getters and Setters
    public String getName() {
        return name;
    }
    
    public void setName(String name) {
        this.name = name;
    }
    
    public int getAge() {
        return age;
    }
    
    public void setAge(int age) {
        if (age > 0) {  // Basic validation
            this.age = age;
        }
    }
    
    // Main method to test the Person class
    public static void main(String[] args) {
        // Creating objects of Person class
        Person person1 = new Person("Alice", 25);
        Person person2 = new Person("Bob", 30);
        
        // Calling methods on the objects
        person1.introduce();
        person2.introduce();
        
        // Modifying object state
        person1.celebrateBirthday();
        person2.setName("Robert");
        
        // Checking the changes
        person1.introduce();
        person2.introduce();
    }
}
Output:
Hello, my name is Alice and I am 25 years old.
Hello, my name is Bob and I am 30 years old.
Happy Birthday! Now I am 26 years old.
Hello, my name is Alice and I am 26 years old.
Hello, my name is Robert and I am 30 years old.

Key OOP Concepts in Java:

  • Class: A blueprint for creating objects. It defines a datatype by bundling data and methods that work on that data.
  • Object: An instance of a class. When a class is defined, no memory is allocated, but when it is instantiated (i.e., an object is created), memory is allocated.
  • Encapsulation: The bundling of data with the methods that operate on that data. It restricts direct access to some of an object's components, which is a means of preventing accidental interference and misuse of the data.
  • Inheritance: The mechanism by which one class can inherit the fields and methods of another class.
  • Polymorphism: The ability of different classes to be treated as instances of the same class through inheritance.
  • Abstraction: The concept of hiding the complex implementation details and showing only the necessary features of an object.

Practice Exercises

Exercise 1: Simple Calculator

Create a simple calculator program that can perform addition, subtraction, multiplication, and division operations on two numbers.

  1. Create a class named Calculator.
  2. Implement methods for each operation: add, subtract, multiply, and divide.
  3. Each method should take two parameters and return the result of the operation.
  4. In the main method, create a Calculator object and test each operation with sample values.
  5. For the divide method, handle the case where the second parameter is zero (division by zero).
Calculator.java 
public class Calculator {
    // Method to add two numbers
    public double add(double a, double b) {
        return a + b;
    }
    
    // Method to subtract two numbers
    public double subtract(double a, double b) {
        return a - b;
    }
    
    // Method to multiply two numbers
    public double multiply(double a, double b) {
        return a * b;
    }
    
    // Method to divide two numbers
    public double divide(double a, double b) {
        if (b == 0) {
            System.out.println("Error: Division by zero is not allowed.");
            return 0;
        }
        return a / b;
    }
    
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        // Test the calculator
        double num1 = 10.0;
        double num2 = 5.0;
        
        System.out.println(num1 + " + " + num2 + " = " + calc.add(num1, num2));
        System.out.println(num1 + " - " + num2 + " = " + calc.subtract(num1, num2));
        System.out.println(num1 + " * " + num2 + " = " + calc.multiply(num1, num2));
        System.out.println(num1 + " / " + num2 + " = " + calc.divide(num1, num2));
        
        // Test division by zero
        System.out.println(num1 + " / 0 = ");
        calc.divide(num1, 0);
    }
}

Exercise 2: Student Grade Tracker

Create a program that tracks student grades and calculates their average.

  1. Create a class named Student with fields for name and an array of grades.
  2. Implement a constructor that initializes the name and creates an empty array for grades.
  3. Implement a method to add a grade to the student's record.
  4. Implement a method to calculate and return the average grade.
  5. Implement a method to display the student's information, including name, all grades, and the average.
  6. In the main method, create a few Student objects, add some grades, and display their information.
Student.java 
import java.util.ArrayList;

public class Student {
    private String name;
    private ArrayList grades;
    
    // Constructor
    public Student(String name) {
        this.name = name;
        this.grades = new ArrayList<>();
    }
    
    // Method to add a grade
    public void addGrade(double grade) {
        grades.add(grade);
    }
    
    // Method to calculate average grade
    public double calculateAverage() {
        if (grades.isEmpty()) {
            return 0.0;
        }
        
        double sum = 0.0;
        for (Double grade : grades) {
            sum += grade;
        }
        
        return sum / grades.size();
    }
    
    // Method to display student information
    public void displayInfo() {
        System.out.println("Student: " + name);
        System.out.print("Grades: ");
        
        if (grades.isEmpty()) {
            System.out.println("No grades yet.");
        } else {
            for (Double grade : grades) {
                System.out.print(grade + " ");
            }
            System.out.println();
            System.out.println("Average: " + calculateAverage());
        }
        System.out.println();
    }
    
    public static void main(String[] args) {
        // Create students
        Student student1 = new Student("Alice");
        Student student2 = new Student("Bob");
        
        // Add grades for Alice
        student1.addGrade(85.5);
        student1.addGrade(92.0);
        student1.addGrade(78.5);
        
        // Add grades for Bob
        student2.addGrade(90.0);
        student2.addGrade(88.5);
        student2.addGrade(95.0);
        
        // Display student information
        student1.displayInfo();
        student2.displayInfo();
    }
}

1. What is the main advantage of Java's "Write Once, Run Anywhere" feature?

It allows Java programs to run faster than programs written in other languages.
It allows Java programs to run on any device with a Java Virtual Machine, regardless of the underlying hardware or operating system.
It allows Java programs to be written once and automatically translated into other programming languages.
It allows Java programs to be written without any syntax errors.

Explanation: Java's "Write Once, Run Anywhere" (WORA) principle means that compiled Java code (bytecode) can run on any platform that has a Java Virtual Machine (JVM), without needing to be recompiled for each specific platform. This platform independence is one of Java's key features.

2. Which of the following is NOT a primitive data type in Java?

int
boolean
char
String

Explanation: String is not a primitive data type in Java; it's a class. The eight primitive data types in Java are: byte, short, int, long, float, double, char, and boolean. String is a reference type that represents a sequence of characters.

3. What is the output of the following code?
int x = 5;
System.out.println(x++ + ++x);

10
11
12
13

Explanation: Let's break this down step by step:

1. x++ is a post-increment operator, which means it uses the current value of x (5) in the expression and then increments x to 6.

2. ++x is a pre-increment operator, which means it first increments x to 7 and then uses this value in the expression.

3. So, the expression becomes 5 + 7 = 12.

4. Which keyword is used to inherit a class in Java?

extends
implements
inherits
super

Explanation: In Java, the extends keyword is used to inherit a class. For example, class Child extends Parent means that the Child class inherits from the Parent class. The implements keyword is used to implement an interface, not to inherit a class.

5. What is the purpose of the JVM in Java?

To compile Java source code into machine code
To provide a set of libraries for Java programs
To execute Java bytecode on different platforms
To optimize Java code for better performance

Explanation: The Java Virtual Machine (JVM) is an abstract computing machine that enables a computer to run Java programs. It executes Java bytecode, which is the compiled form of Java source code. The JVM is the cornerstone of the Java platform's "Write Once, Run Anywhere" principle, as it allows the same bytecode to run on any device with a compatible JVM, regardless of the underlying hardware or operating system.
