💰 Greedy Algorithms

A Greedy Algorithm builds up a solution piece by piece, always choosing the next piece that offers the most immediate benefit. It doesn’t always provide an optimal solution, but when it does, it's usually the most efficient approach.

🧠 Key Idea

At each step, choose the option that looks best at that moment. Make a local optimal choice, hoping it leads to a global optimum.

🔍 When to Use Greedy

  • Problems where future consequences don’t affect the present decision.
  • Optimization problems (maximize or minimize).
  • When the greedy choice property and optimal substructure are satisfied.

⚙️ Popular Greedy Algorithms

  1. Activity Selection
  2. Huffman Coding
  3. Job Sequencing with Deadline
  4. Fractional Knapsack
  5. Dijkstra’s Algorithm
  6. Prim’s and Kruskal’s Algorithm

💡 Example: Activity Selection (C++)


bool activityCompare(pair a, pair b) {
    return a.second < b.second;
}
void printActivities(vector> activities) {
    sort(activities.begin(), activities.end(), activityCompare);
    int lastEnd = activities[0].second;
    cout << "Selected: (Start: " << activities[0].first << ", End: " << lastEnd << ")\n";
    for (int i = 1; i < activities.size(); i++) {
        if (activities[i].first >= lastEnd) {
            cout << "Selected: (Start: " << activities[i].first << ", End: " << activities[i].second << ")\n";
            lastEnd = activities[i].second;
        }
    }
}

🌐 Real-World Applications

  • Scheduling (CPU Jobs, Railway Platforms)
  • Minimum Spanning Tree (Kruskal/Prim)
  • Network Routing
  • Data Compression (Huffman Coding)
Pro Tip: Prove greedy choice property and optimal substructure before using greedy. Otherwise, the result may not be optimal.

📝 Key Characteristics

  • No backtracking or reconsideration
  • Fast and space-efficient
  • Not guaranteed to be optimal in all problems
