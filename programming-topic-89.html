⚡ Greedy Algorithms

A Greedy Algorithm builds up a solution piece by piece, always choosing the next piece that offers the most immediate benefit. It doesn't reconsider its choices, unlike Dynamic Programming or Backtracking.

🔍 Core Idea

  • Make a **locally optimal** choice at each step
  • Hopes that this leads to a **globally optimal** solution
  • Faster than exhaustive techniques, but works only for certain problems

📘 Greedy Algorithm Examples

  1. Activity Selection Problem
  2. Fractional Knapsack Problem
  3. Huffman Encoding
  4. Job Sequencing Problem
  5. Prim’s and Kruskal’s MST Algorithms
  6. Dijkstra’s Shortest Path Algorithm

💼 Real-World Use Cases

  • Network routing protocols (Dijkstra)
  • Data compression (Huffman Coding)
  • Project scheduling
  • Task prioritization systems

📌 Example: Activity Selection

Given start and end times of N activities, select the maximum number of non-overlapping activities.


void activitySelection(int start[], int end[], int n) {
  vector<pair<int,int>> activities;
  for (int i = 0; i < n; i++)
    activities.push_back({end[i], start[i]});
  
  sort(activities.begin(), activities.end());
  
  cout << "Selected Activities:\n";
  int count = 1, lastEnd = activities[0].first;
  cout << "(" << activities[0].second << "," << lastEnd << ")\n";
  
  for (int i = 1; i < n; i++) {
    if (activities[i].second >= lastEnd) {
      cout << "(" << activities[i].second << "," << activities[i].first << ")\n";
      lastEnd = activities[i].first;
      count++;
    }
  }
}

⚖️ Greedy vs Dynamic Programming

Aspect Greedy Dynamic Programming
Approach Local optimization Global optimization
Backtracking Required? No Sometimes
Performance Faster Slower but accurate
Use Cases When greedy works optimally Overlapping subproblems

🧠 Key Tips to Master Greedy Algorithms

  • Try proving the greedy choice is optimal (Greedy-choice property)
  • Check if the problem has **optimal substructure**
  • Simulate the problem manually to observe patterns
  • Sort wisely — many greedy problems require sorted inputs
