🔁 Dynamic Programming (DP)

Dynamic Programming (DP) is an optimization technique used to solve complex problems by breaking them into smaller subproblems, solving each subproblem only once, and storing its result for future use (memoization or tabulation).

🔍 Why DP?

  • Overcomes **overlapping subproblems** issue seen in recursion.
  • Improves efficiency by **avoiding repeated computation**.
  • Uses either **top-down (memoization)** or **bottom-up (tabulation)** approach.

📘 Classic DP Problems

  1. Fibonacci Series (Optimized)
  2. 0/1 Knapsack Problem
  3. Longest Common Subsequence (LCS)
  4. Edit Distance
  5. Matrix Chain Multiplication
  6. Coin Change Problem

🧪 Example: Fibonacci with Memoization (C++)


int fib(int n, vector<int> &dp) {
    if(n <= 1) return n;
    if(dp[n] != -1) return dp[n];
    return dp[n] = fib(n-1, dp) + fib(n-2, dp);
}
int main() {
    int n = 10;
    vector<int> dp(n+1, -1);
    cout << fib(n, dp);
}

📊 Memoization vs Tabulation

Memoization Tabulation
Top-down approach Bottom-up approach
Uses recursion + caching Uses loops to build solution
Easier to write, but can use more stack space Faster in space-critical scenarios

🏆 Benefits

  • Improves efficiency from exponential to polynomial
  • Ensures optimal substructure is fully utilized
  • Handles wide range of optimization problems

🌐 Real-World Use Cases

  • Spell checkers (Edit Distance)
  • Data compression algorithms
  • Bioinformatics (DNA sequence alignment)
  • Stock market prediction models
  • AI planning and games (minimax with memoization)
💡 Tip: Identify overlapping subproblems and optimal substructure to apply DP correctly.
