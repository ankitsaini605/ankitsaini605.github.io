🔙 Backtracking

Backtracking is an algorithmic paradigm used for solving constraint satisfaction problems by building a solution incrementally and removing those solutions that fail to satisfy the constraints of the problem.

🧠 Key Concept

Try out all possibilities. If a path leads to a dead-end, **go back (backtrack)** and try another possibility. Think of it like exploring a maze.

📌 Common Problems Using Backtracking

  • N-Queens Problem
  • Sudoku Solver
  • Rat in a Maze
  • Subset Sum
  • Graph Coloring
  • Word Search

🧾 Code Example: N-Queens Problem (4x4) in C++


bool isSafe(int board[4][4], int row, int col) {
    for (int i = 0; i < col; i++)
        if (board[row][i]) return false;

    for (int i=row, j=col; i>=0 && j>=0; i--, j--)
        if (board[i][j]) return false;

    for (int i=row, j=col; i<4 && j>=0; i++, j--)
        if (board[i][j]) return false;

    return true;
}

bool solveNQUtil(int board[4][4], int col) {
    if (col >= 4) return true;
    for (int i = 0; i < 4; i++) {
        if (isSafe(board, i, col)) {
            board[i][col] = 1;
            if (solveNQUtil(board, col + 1)) return true;
            board[i][col] = 0;
        }
    }
    return false;
}

🎯 Features of Backtracking

  • Recursive in nature
  • Exponential time complexity in worst case
  • Efficient for small input sizes or heavily constrained problems

🌐 Real-Life Use Cases

  • AI Game Solving (like Sudoku)
  • Constraint Programming
  • Path Finding in Games or Maps
Pro Tip: Always try to add **pruning techniques** to reduce the number of recursive calls and optimize performance.
