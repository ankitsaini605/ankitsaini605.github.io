💰 Greedy Algorithms - Concept & Implementation

Greedy algorithms are a class of algorithms that make the optimal choice at each step with the hope that the global optimum will be reached. These are simple, intuitive, and often surprisingly effective.

🔍 Key Characteristics of Greedy Algorithms

  • Greedy Choice Property: A global optimum can be arrived at by selecting a local optimum.
  • Optimal Substructure: The problem can be broken into subproblems which can be solved optimally.

⚙️ Famous Greedy Problems

  1. Activity Selection
  2. Fractional Knapsack
  3. Huffman Encoding
  4. Minimum Spanning Tree (Prim’s & Kruskal’s)
  5. Dijkstra’s Shortest Path

🧠 Example: Activity Selection Problem

// C++ Example
bool compare(Activity a, Activity b) {
    return a.finish < b.finish;
}

void activitySelection(vector activities) {
    sort(activities.begin(), activities.end(), compare);
    int lastFinish = 0;
    for (auto act : activities) {
        if (act.start >= lastFinish) {
            cout << act.name << " ";
            lastFinish = act.finish;
        }
    }
}

📌 Advantages

  • Faster than dynamic programming or exhaustive search
  • Easy to implement and understand
  • Great for problems with clear greedy property

⚠️ When NOT to Use Greedy

  • If local optima don’t lead to global optimum
  • If multiple combinations need to be tested

🛠️ Real-Life Applications

  • File compression algorithms (like Huffman)
  • Job scheduling in OS
  • Travel route optimization
Pro Tip 🧮: Prove greedy correctness with mathematical induction or by comparing to dynamic programming results.
