🧠 Dynamic Programming (DP)

Dynamic Programming is a powerful technique used to solve complex problems by breaking them down into simpler subproblems. It’s a key topic for technical interviews and competitive programming.

📚 What is Dynamic Programming?

DP is an optimization over recursion. It stores the results of subproblems to avoid recomputation — this is called **memoization** or **tabulation**.

🔁 Memoization (Top-Down DP)


int fib(int n, vector<int>& dp) {
  if (n <= 1) return n;
  if (dp[n] != -1) return dp[n];
  return dp[n] = fib(n - 1, dp) + fib(n - 2, dp);
}

📊 Tabulation (Bottom-Up DP)


int fib(int n) {
  vector<int> dp(n + 1);
  dp[0] = 0; dp[1] = 1;

  for (int i = 2; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
  }

  return dp[n];
}

🎯 Common DP Patterns

  • Fibonacci Sequence
  • 0/1 Knapsack
  • Longest Common Subsequence (LCS)
  • Matrix Path Problems
  • Coin Change
  • Rod Cutting
  • Palindrome Partitioning

🧪 0/1 Knapsack (Classic DP Problem)


int knapsack(int wt[], int val[], int W, int n) {
  vector<vector<int>> dp(n+1, vector<int>(W+1, 0));

  for (int i = 1; i <= n; i++) {
    for (int w = 1; w <= W; w++) {
      if (wt[i-1] <= w)
        dp[i][w] = max(val[i-1] + dp[i-1][w - wt[i-1]], dp[i-1][w]);
      else
        dp[i][w] = dp[i-1][w];
    }
  }
  return dp[n][W];
}
Tip: Always identify overlapping subproblems and optimal substructure before applying DP.

⏱️ Time Complexities

  • Fibonacci (with memoization): O(n)
  • Knapsack: O(n * W)
  • LCS: O(m * n)
