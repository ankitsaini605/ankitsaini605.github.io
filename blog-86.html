⚡ Greedy Algorithms

A Greedy Algorithm makes locally optimal choices at each step with the hope of finding a global optimum. It works best for problems that have the **Greedy Choice Property** and **Optimal Substructure**.

🔍 When to Use Greedy

  • When each local choice leads to a global solution.
  • When the problem can be proven to have the greedy property (e.g., exchange argument).
  • When you're solving optimization problems efficiently.

📘 Famous Greedy Problems

  1. Activity Selection Problem
  2. Fractional Knapsack
  3. Huffman Coding
  4. Kruskal's Minimum Spanning Tree
  5. Prim's Algorithm
  6. Job Sequencing with Deadlines

🧪 Example: Activity Selection


// Sort activities by end time
bool comp(Activity a1, Activity a2) {
  return a1.end < a2.end;
}

void activitySelection(vector<Activity>& activities) {
  sort(activities.begin(), activities.end(), comp);
  cout << "Selected: " << activities[0].id;
  int lastEnd = activities[0].end;

  for (int i = 1; i < activities.size(); i++) {
    if (activities[i].start >= lastEnd) {
      cout << ", " << activities[i].id;
      lastEnd = activities[i].end;
    }
  }
}

🧾 Example: Fractional Knapsack

Greedy approach works in the fractional knapsack problem by picking the item with the highest value/weight ratio first.

📊 Greedy vs Dynamic Programming

Aspect Greedy Dynamic Programming
Approach Top-down with local choice Bottom-up or memoized recursion
Time Efficiency Very fast (O(n log n) or better) Moderate to high (O(n²) or more)
Use Cases Simple, optimization problems Complex problems with overlapping subproblems

🧠 Tips for Interviews

  • Check if local optimal → global optimal.
  • Use sorting wisely (by value/weight, end time, cost, etc.).
  • Always try both Greedy and DP if unsure.
