📐 Dynamic Programming (DP)

Dynamic Programming is an optimization technique used for solving complex problems by breaking them down into simpler subproblems and storing the results to avoid redundant computations.

🧠 Why Use DP?

  • Reduces time complexity in recursive problems
  • Efficient for overlapping subproblems
  • Great for optimization problems like knapsack, LCS, LIS, etc.

📂 Two Main Techniques

  • Top-Down (Memoization): Recursion + Cache
  • Bottom-Up (Tabulation): Iterative + DP Table

🔥 Classic DP Problems

  1. 0/1 Knapsack Problem
  2. Longest Common Subsequence (LCS)
  3. Longest Increasing Subsequence (LIS)
  4. Matrix Chain Multiplication
  5. Coin Change
  6. Edit Distance

🧪 Example: Fibonacci with DP


// Memoization (Top-Down)
int fib(int n, vector<int>& dp) {
  if (n <= 1) return n;
  if (dp[n] != -1) return dp[n];
  return dp[n] = fib(n - 1, dp) + fib(n - 2, dp);
}

// Tabulation (Bottom-Up)
int fib(int n) {
  vector<int> dp(n + 1);
  dp[0] = 0; dp[1] = 1;
  for (int i = 2; i <= n; i++)
    dp[i] = dp[i - 1] + dp[i - 2];
  return dp[n];
}

📊 Comparison: Memoization vs Tabulation

Aspect Memoization Tabulation
Approach Top-down (recursive) Bottom-up (iterative)
Function Calls Recursive stack used No recursion
Performance May be slower due to stack Faster in tight loops

📚 Tips for Mastering DP

  • Identify overlapping subproblems
  • Draw a recursion tree or DP table
  • Write base cases clearly
  • Optimize space (sometimes 1D array is enough)

🛠️ Real-World Applications

  • Spell Checkers (Edit Distance)
  • Text Prediction (LCS)
  • Route Optimization (Matrix Chain)
  • Stock Profit Calculation
