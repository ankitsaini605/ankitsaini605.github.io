⚡ Greedy Algorithms

A **Greedy Algorithm** makes the locally optimal choice at each step with the hope of finding the global optimum. These algorithms work in problems where choosing local optimums leads to the final solution.

📌 Characteristics of Greedy Algorithms

  • Greedy Choice Property
  • Optimal Substructure
  • No Backtracking Required

🎯 Famous Greedy Problems

  • Activity Selection
  • Fractional Knapsack
  • Job Scheduling
  • Huffman Coding
  • Minimum Number of Coins
  • Gas Station Problem

✅ Example: Activity Selection


bool activitySelection(vector<pair<int, int>> activities) {
  sort(activities.begin(), activities.end(), [](auto &a, auto &b) {
    return a.second < b.second;
  });

  int count = 1;
  int end = activities[0].second;

  for (int i = 1; i < activities.size(); i++) {
    if (activities[i].first >= end) {
      count++;
      end = activities[i].second;
    }
  }

  return count;
}

💰 Fractional Knapsack (Greedy)

Unlike 0/1 knapsack, we can take fractions of items. 


bool compare(Item a, Item b) {
  double r1 = (double)a.value / a.weight;
  double r2 = (double)b.value / b.weight;
  return r1 > r2;
}

double fractionalKnapsack(int W, Item arr[], int n) {
  sort(arr, arr + n, compare);

  double finalValue = 0.0;

  for (int i = 0; i < n; i++) {
    if (arr[i].weight <= W) {
      W -= arr[i].weight;
      finalValue += arr[i].value;
    } else {
      finalValue += arr[i].value * ((double)W / arr[i].weight);
      break;
    }
  }

  return finalValue;
}

🔠 Huffman Coding (Greedy + Trees)

Builds the most efficient encoding tree based on character frequency using a min-heap. Used in data compression.

Pro Tip: Always verify if greedy choice guarantees global optimality. Else, use Dynamic Programming.

⏱️ Time Complexities

  • Activity Selection: O(n log n)
  • Fractional Knapsack: O(n log n)
  • Huffman Coding: O(n log n)
