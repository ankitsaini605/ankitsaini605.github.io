🚀 Dynamic Programming (DP)

**Dynamic Programming (DP)** is an advanced algorithmic technique used to optimize problems that have **overlapping subproblems** and **optimal substructure**. Instead of solving the same subproblem multiple times, DP stores results to avoid redundant computations.

📘 Key Techniques

  • Memoization (Top-Down): Recursion + caching results.
  • Tabulation (Bottom-Up): Iterative approach storing solutions in a table.

🔢 Example 1: Fibonacci Number (Memoization)


int fib(int n, vector<int>& dp) {
  if (n <= 1) return n;
  if (dp[n] != -1) return dp[n];
  return dp[n] = fib(n - 1, dp) + fib(n - 2, dp);
}

📊 Example 2: 0/1 Knapsack Problem


int knapsack(vector<int>& wt, vector<int>& val, int W, int n) {
  vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));
  for (int i = 1; i <= n; i++) {
    for (int w = 1; w <= W; w++) {
      if (wt[i - 1] <= w)
        dp[i][w] = max(val[i - 1] + dp[i - 1][w - wt[i - 1]], dp[i - 1][w]);
      else
        dp[i][w] = dp[i - 1][w];
    }
  }
  return dp[n][W];
}

🧬 Real-World Problems Solved by DP

  • ✔️ Longest Common Subsequence (LCS)
  • ✔️ Longest Increasing Subsequence (LIS)
  • ✔️ Edit Distance (Levenshtein Distance)
  • ✔️ Matrix Chain Multiplication
  • ✔️ Minimum Cost Path
  • ✔️ Palindromic Substrings

⚙️ Tabulation vs Memoization

Feature Memoization Tabulation
Approach Top-Down Bottom-Up
Recursion Yes No
Space Optimization Harder More controllable

📚 Interview Tips

  • 🧠 Break the problem into subproblems.
  • 🧩 Identify overlapping subproblems.
  • 🧮 Use a memo table or 2D DP array when necessary.
  • 🔁 Start with recursion, then convert to DP.
