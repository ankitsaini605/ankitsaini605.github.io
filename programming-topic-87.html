🔁 Backtracking Algorithm

Backtracking is a technique for solving problems incrementally, one step at a time, removing solutions that fail to satisfy the constraints. It’s based on **recursion + trial and error**.

💡 When to Use Backtracking

  • To explore all possibilities (combinatorics).
  • To solve constraint satisfaction problems.
  • When brute force is too inefficient and pruning is possible.

📘 Famous Problems

  1. N-Queens Problem
  2. Sudoku Solver
  3. Rat in a Maze
  4. Subset/Permutation Generation
  5. Word Search in Matrix

🧪 Example: N-Queens Problem


bool isSafe(int board[][N], int row, int col) {
  // Check for queen in this row on the left side
  ...
}

bool solveNQUtil(int board[][N], int col) {
  if (col >= N)
    return true;

  for (int i = 0; i < N; i++) {
    if (isSafe(board, i, col)) {
      board[i][col] = 1;
      if (solveNQUtil(board, col + 1))
        return true;
      board[i][col] = 0; // BACKTRACK
    }
  }

  return false;
}

📊 Backtracking vs Brute Force

Aspect Backtracking Brute Force
Search Strategy Recursive pruning of invalid paths Check all possibilities blindly
Efficiency Much better when pruning works Usually exponential and slow
Use Case Combinatorial + constraint problems Small inputs or exhaustive testing

🧠 Backtracking Template (Pseudocode)


function backtrack(path, options):
  if path is a complete solution:
    output(path)
    return
  for option in options:
    if isValid(option):
      path.add(option)
      backtrack(path, options)
      path.remove(option)  // undo
