💡 Greedy Algorithm - Making Locally Optimal Choices

Greedy algorithms build up a solution piece by piece, always choosing the next piece that offers the most immediate benefit (locally optimal), without worrying about future consequences.

🧠 Concept Behind Greedy

  • At each step, pick the best option currently available.
  • Never backtrack or revise previous decisions.
  • Works well for problems with greedy-choice property and optimal substructure.

🔧 Structure of a Greedy Algorithm


1. Sort the input based on a specific strategy.
2. Initialize result = 0 or empty set.
3. Iterate through elements:
    - Pick the best local option.
    - If feasible, include it in the solution.
4. Return the result.

✅ Common Greedy Problems

  1. Fractional Knapsack
  2. Activity Selection Problem
  3. Huffman Coding
  4. Job Sequencing with Deadlines
  5. Minimum Spanning Tree (Prim’s & Kruskal’s)
  6. Dijkstra’s Algorithm for Shortest Path

📘 Example: Activity Selection Problem in C++


bool activityCompare(Activity a1, Activity a2) {
    return (a1.finish < a2.finish);
}
void selectActivities(vector<Activity> activities) {
    sort(activities.begin(), activities.end(), activityCompare);
    cout << "Selected Activities: ";
    int lastFinish = activities[0].finish;
    cout << "(" << activities[0].start << "," << activities[0].finish << ")";
    for (int i = 1; i < activities.size(); i++) {
        if (activities[i].start >= lastFinish) {
            cout << ", (" << activities[i].start << "," << activities[i].finish << ")";
            lastFinish = activities[i].finish;
        }
    }
}

🏆 Advantages

  • Simple and fast implementation
  • Efficient for certain problem types
  • Often optimal for real-time systems

⚠️ Limitations

  • Does not always guarantee the global optimum
  • Fails for problems requiring future foresight (like 0/1 Knapsack)

🌍 Real-World Applications

  • Network routing algorithms
  • Resource allocation in operating systems
  • Scheduling systems (airlines, sports, events)
  • Data compression (Huffman coding)
💡 Pro Tip: Before choosing greedy, check if your problem has the **greedy-choice property** and **optimal substructure**.
