🔄 Selection Sort Algorithm

A simple comparison-based algorithm for sorting arrays in ascending order.

📘 What is Selection Sort?

Selection Sort is a basic sorting technique where the algorithm repeatedly finds the minimum element from the unsorted part of the array and moves it to the beginning. It is intuitive but not very efficient for large datasets.

📌 Key Concepts:

  • Divides array into sorted and unsorted parts
  • Selects the smallest element and places it at the correct position
  • Comparison-based: O(n²) time complexity

🐍 Python Code Example:


def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i+1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr

arr = [64, 25, 12, 22, 11]
sorted_arr = selection_sort(arr)
print("Sorted Array:", sorted_arr)

💻 C++ Code Example:


#include <iostream>
using namespace std;

void selectionSort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {
        int min_idx = i;
        for (int j = i+1; j < n; j++)
            if (arr[j] < arr[min_idx])
                min_idx = j;
        swap(arr[i], arr[min_idx]);
    }
}

int main() {
    int arr[] = {64, 25, 12, 22, 11};
    int n = sizeof(arr)/sizeof(arr[0]);
    selectionSort(arr, n);
    cout << "Sorted array: ";
    for (int i = 0; i < n; i++)
        cout << arr[i] << " ";
    return 0;
}

⏱ Time & Space Complexity:

  • Best Case: O(n²)
  • Average Case: O(n²)
  • Worst Case: O(n²)
  • Space: O(1)

✅ Advantages:

  • Simple to implement
  • Performs well on small data

⚠️ Disadvantages:

  • Inefficient on large lists
  • Does not adapt or become faster on sorted arrays
