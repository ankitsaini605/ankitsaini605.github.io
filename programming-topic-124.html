📌 What is Dynamic Programming?

Dynamic Programming (DP) is a technique to solve complex problems by breaking them down into simpler subproblems and solving each subproblem only once, storing the results for future use (memoization or tabulation).

💡 When to Use DP?

  • Overlapping Subproblems
  • Optimal Substructure

🔁 Top-Down Approach (Memoization) - Python


def fib(n, memo={}):
    if n in memo: return memo[n]
    if n <= 1: return n
    memo[n] = fib(n-1, memo) + fib(n-2, memo)
    return memo[n]

print(fib(10))  # Output: 55

⬇️ Bottom-Up Approach (Tabulation) - Python


def fib_tab(n):
    dp = [0, 1]
    for i in range(2, n+1):
        dp.append(dp[i-1] + dp[i-2])
    return dp[n]

print(fib_tab(10))  # Output: 55

int fib(int n, unordered_map& memo) {
    if (memo.count(n)) return memo[n];
    if (n <= 1) return n;
    memo[n] = fib(n-1, memo) + fib(n-2, memo);
    return memo[n];
}

int main() {
    unordered_map memo;
    cout << fib(10, memo);  // Output: 55
}

💡 Tip: Use space optimization when you only need the last few computed values instead of storing the full array.
